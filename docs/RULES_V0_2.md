# 游戏规则 v0.2（与当前实现对齐修订）

与实现对齐的权威玩法说明；测试用例须引用本文。

> **修订说明**：条文已与仓库当前行为一致——开局为**选 3 张牌**；组牌含**连对、飞机**；装配间隔在**本轮最后一组打出后**开始（不等弹道命中）。**§8 局外金币、§9 胜利条件**仍为规划，运行时未实现，后续版本再接。

## 实现状态（摘要）

- **已接入**：开局选牌、牌组循环与间隔、`GroupDetector` 组牌（含扩展牌型）、`PlayEffectResolver`（关键张 2/8/10 + 回落 `EffectResolver`）、`CombatEffectRunner` 命令执行、与自动攻击/弹道联动。  
- **未实现**：§8 局外金币、§9 胜利计时、遗物与 RULES §5 中「遗物」档位的完整管线；§10 与花色 §6 的二次叠加（当前关键张组不接花色对弹道改写）。详见 `测试与进度.md`。

## 1. 开局

单局开局时，玩家在开局界面 **依序选择 3 张牌**，构成**有序**初始牌组（与 `RunScene`、`CardSelectUI` 一致）。若未来改为牌池随机发牌，须同步修订本文档。

## 2. 出牌循环

从左到右自动出牌，基础间隔 **0.5s**（可被升级/遗物修改）。每步从当前指针做组牌并打出。

当本轮指针走完牌组（**最后一组牌已打出**）后，进入 **装配间隔 2.0s**（可修改），再从最左继续。

**说明**：装配计时以逻辑上「本步组牌打出完成」为界，**不要求**等待弹道命中、伤害飘字等表现层全部结束（与 `CardRuntime` 一致）。

## 3. 组牌类型

以下均为合法组牌；无法构成任一类型则本步为 **单张组**。

- **对子、三条、四条**。  
- **顺子**：最短长度 **3**；**A 不可当 1**；允许 **Q-K-A**。  
- **连对**：至少 **2** 组**连续点数**的对子（总张数 ≥4 且为偶数；最小连对数等见 `GroupDetector` 常量）。  
- **飞机**：至少 **2** 组**连续点数**的三条（总张数 ≥6 且为 3 的倍数；最小组数见 `GroupDetector` 常量）。  

在多种牌型同时成立时，取 **张数最长** 的一组；张数相同时按实现内定义的**牌型优先级**取组（见 `GroupDetector`）。

## 4. 扫描与优先级

始终从左到右；在当前起点取 **最长合法组**；打出后指针 **跳过整组**；每张牌在一轮遍历中 **不可** 属于多个组。

## 5. 多效果叠加

同一组满足多种条件时效果**同时生效**。建议结算顺序（实现固定）：  
① 基础牌效果 → ② 组牌效果 → ③ 本次攻击花色 → ④ 同花大幅强化 → ⑤ 全局被动（花色阈值、遗物等）。

## 6. 花色 — 本次攻击

对象为**当前打出的一组**的全部张数。某花色张数 **≥2** 触发该花色本次效果；若触发且**整组同花色**，该花色效果**大幅强化**。

## 7. 花色 — 全局（构筑）

**每个花色独立**统计牌组中该花色张数；达 **3 / 6 / 9 / 12** 解锁对应全局档位。

## 8. 局外金币（规划，未实现）

**金币 = 牌价值 × 存活时间系数**；牌价值按组牌统计，单张打出按单张组计。  
> 当前版本无运行时逻辑；后续版本规划。

## 9. 胜利条件（规划，未实现）

单局生存 **20 分钟** 胜利（关卡系统可后接）。  
> 当前版本无运行时逻辑；后续版本规划。

## 10. 关键张点数效果（基础管线，与 `设计与架构.md` §7 对齐）

以下均要求：打出的**整组**为**同一 `CardResource.rank`**（用牌面点数，**非** `get_rank_value()` 的 A=14 规则），且 `GroupDetector` 给出的 `group_type` 与张数一致。满足时走 **关键张解析**，**不再**对该组套用旧版 `EffectResolver` 的弹道数值链；**花色本地 §6 / 全局 §7 对该组弹道暂不二次叠乘**（治疗类命令不受影响以外的部分无额外花色修正），后续版本再与 §5 完全对齐。

| 条件 | 效果 |
|------|------|
| 单张 **2** | 普通 **双发** 弹道（两发）；多发时经 **`ProjectileToolSingleton` 并行布局**（共向、侧向错开起点） |
| **对子 2** | **四发** 弹道（两对双发语义：2×2 发），并行布局 |
| **三条 2** | **六发** 弹道，并行布局 |
| **四条 2** | **双道短激光**（多段伤害） |
| 单张 **8** | **1** 枚红色闪烁专用弹道（**不复用**点数 2 的弹道预制体）；自玩家处按**角度扇**取发射向，**直线**飞行，近敌或超时后在落点 **圆形爆炸** |
| **对子 8** | **2** 枚上述弹道，**角度扇**均分扫角；每枚各自命中后爆炸，爆炸 **范围增大**（与实现对齐） |
| **三条 8** | **3** 枚弹道，**角度扇**；每枚命中后爆炸，爆炸 **伤害翻倍**（与实现对齐） |
| **四条 8** | **4** 枚弹道，**角度扇**；每枚命中后爆炸（伤害翻倍）；**最后一枚**命中点额外遗留 **灼烧地面** 若干秒 |
| 单张 **10** | **回复最大生命 10%** |
| **对子 10** | **回复最大生命 30%** |
| **三条 10** | **回复最大生命 50%** 且 **无敌 2 秒** |
| **四条 10** | **回复最大生命 100%** 且 **无敌 5 秒** |

治疗与无敌在 **`CombatEffectRunner` 逻辑相**执行；弹道、爆炸、激光、灼地在 **表现相**执行。点数 **8** 的弹道枚数与组内张数一致，**后续若接花色/遗物「强化」可增加枚数**，仍以本表语义为纲。具体数值常量见 `scripts/core/combat_tuning.gd` 中 `RANK_*` 前缀项。
