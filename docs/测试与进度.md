## 二、冒烟测试清单

合并或发布前至少过一遍；未实现模块可跳过。

### Godot MCP 验证（测试环节规范）

对本仓库做**冒烟测试**或**实现完成后的自检**时，若 **Godot MCP Pro 已在当前客户端连接成功**（Godot 打开本项目、插件启用、MCP 服务可用），应**优先通过 Godot MCP** 做验证，而不是依赖终端里能否直接调用 `godot` 命令行。前置条件、工具列表与局限见 `docs/开发环境_Godot_MCP.md` **§6**。

**推荐最小闭环**（与 `开发环境` §6.2 一致；Agent 交付前在 MCP 可用时应执行）：

1. 改动脚本后：使用 **`validate_script`**（或等价工具）确认目标 `.gd` 无语法问题。  
2. **`play_scene`**：以主场景或当前验收场景启动运行实例。  
3. 运行中：**`get_editor_errors`** 必查；按需使用 **`get_game_screenshot`**、`get_game_scene_tree`、`get_game_node_properties` 等核对画面与运行时状态。  
4. **`stop_scene`**：结束运行，避免与下一次 `play_scene` 冲突。  

**边界**：MCP 用于「跑起来 + 看报错/看状态」，**不替代**下文「三、卡牌逻辑测试用例」及 **`详细设计.md` 的规则与数值核对**。未连接 MCP 时，仍以 Godot 编辑器手动运行或工作区任务（见 `开发环境` §3–§4）完成冒烟。

### H 菜单与 MCP 快测（同点数 2 · 必做其一）

改动 **`scripts/cards/`**、**`scripts/combat/`** 或同点数弹道表现时，交付前须在 **① H 菜单人工快测** 与 **② Godot MCP Pro 脚本断言** 中**至少完成一种**（推荐两者都做），并确认**编辑器无新增错误**。

## 三、卡牌逻辑测试用例（与 `详细设计.md` 及实现对照）

### 顺序与分组

1. 起点为 `Q, K, A` → 合法顺子，长度 3。  
2. 起点为 `A, 2, 3` → 非法顺子 → 退化为单张组。  
3. `7,7,7,7` → 四条，长度 4。  
4. `5,5,6,7,8` 从索引 0：以「最长合法组」为准；期望须在实现注释中写死（顺子 `5,6,7,8` 或对子优先等）。  
5. 打出一组后，下一判定从**紧邻下一张**开始，不得跨组复用牌。  

### 出牌循环

6. 整副牌本轮打完（最后一组打出）→ 进入装配/重组计时（与 `CardRuntime` 一致；**不**以弹道命中为界）。  
7. 同一次遍历内步与步之间遵守基础出牌间隔。  

### 花色 — 本次攻击

8. 一组中某花色张数 ≥2 → 触发该花色本次效果。  
9. 满足花色阈值且**整组同花色** → 本次花色效果强化。  
10. 单张组不触发「多张才触发」的本地花色阈值类效果（按具体字段实现）。  

### 花色 — 全局

11. 全牌组内心张数达 3/6/9/12 时分别解锁对应档位。  
12. 各花色阈值**独立**：增加梅花档不得改变红心档。  

### 同点数专属（先行 2/8/10，管线）

13. 单张 `rank=2` 打出 → 日志 `play_plan` 含调试标签 `same_rank_exclusive`，且生成 **2** 发弹道。  
14. 对子 2 → **4** 发弹道；三条 2 → **10** 发（`rank2_volley_count_triple`）；四条 2 → **20** 发随机多发 + 穿透链（`RANK2_FOUR_VOLLEY`，非激光）。  
15. 单张 8 → **1** 枚红色闪烁弹道（**角度扇**直线飞行），近敌或超时爆炸；对子/三条/四条枚数与扇角均分；四条最后一枚命中点有灼地。点数 **2** 多发为 **并行** 错起点。8 弹道**非**点数 2 预制体。  
16. 单张 10 → 生命上升约 **10%**；对子 10 → 约 **30%**；三条 10 → 约 **50%** 且短时内不受敌接触伤害；四条 10 → 回满或近满且无敌更长。  
17. 非 2/8/10 的组牌仍走旧数值链转弹道（`legacy_effect_resolver` 标签）。
