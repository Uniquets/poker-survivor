# 文档索引

本文档目录按“权威文档、开发验证、资料归档”分层维护。新增文档前优先判断能否并入现有文档，避免同一规则在多个地方重复。

## 权威顺序

1. `详细设计.md`：唯一玩法与内容规格权威，包含系统边界、牌型、点数效果、花色、敌人、遗物、Boss、关卡等。
2. `工程规范.md`：代码边界、目录职责、Agent 交付与 Godot MCP 验证要求。
3. `测试与进度.md`：任务看板、冒烟清单、卡牌逻辑用例。验证用例必须服从 `详细设计.md`。
4. `.cursor/rules/cursor-project.mdc`：编辑器/Agent 侧执行规则，内容应与上面三份文档保持一致。

## 资料目录

| 路径 | 用途 |
|------|------|
| `开发环境_Godot_MCP.md` | Godot、Cursor/Codex 与 Godot MCP Pro 的开发验证环境 |

| `art/AI美术素材生成提示词.md` | AI 美术生成提示词与流程 |

## 维护原则

- 玩法规则只写进 `详细设计.md`。
- 工程约束只写进 `工程规范.md`。
- 可执行验收项只写进 `测试与进度.md`。
- README 只做入口导航，不承载玩法细则。
