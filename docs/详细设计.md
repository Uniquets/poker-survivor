## 1. 游戏定位与体验支柱

### 1.1 一句话定位

**俯视角类幸存者战斗** + **有序扑克牌组驱动的自动攻击与构筑**：玩家专注走位与生存，伤害节奏由牌组顺序与组牌结果决定，通过入牌、花色阈值与（规划中的）遗物形成长期 Build。

### 1.2 体验支柱（设计约束）


| 支柱      | 玩家应感受到的                     | 系统承担方式                  |
| ------- | --------------------------- | ----------------------- |
| **可读性** | 能预判「下一步会打出哪一组、 roughly 强不强」 | 牌组 UI、出牌高亮、组牌结果可日志/飘字反馈 |
| **技巧感** | 排序与选牌能改变战局，而非纯随机            | 开局选牌、手牌排序、升级入牌改变扫描序列    |
| **压迫感** | 典型幸存者敌潮与走位压力                | 刷新、碰撞伤害、弹道与近身威胁         |
| **成长感** | 单局内变强路径清晰                   | 经验、升级、入牌、花色全局档、（规划）遗物   |


### 1.3 目标受众与单局时长

- 熟悉幸存者或 ARPG 的玩家可 **30 秒内**理解移动 + 「牌在自动出」。  
- 单局目标时长由 **章节与难度** 配置（具体见第二部），与工程基线可分别迭代。

## 2. 核心循环与胜负条件

### 2.1 微观循环（战斗中）

```
走位生存 → 牌组自动从左到右出牌 → 组牌判定 → 效果解析 → 弹道/治疗/无敌等执行
		 → 敌人掉落经验 / 造成伤害 → 升级与入牌机会 → 牌组变化 → 循环
```

### 2.2 中观循环（单局）

开局构筑初始牌组 → 持续战斗与成长 → 角色/牌组强度上升应对升高难度 → **死亡结束**或（规划中）**达成胜利条件**。

### 2.3 胜负与结算（系统层）


| 状态        | 系统职责                                               |
| --------- | -------------------------------------------------- |
| 玩家生命归零    | 结束本局；可展示摘要与再开流程                                    |
| 达成关卡目标    | 进入胜利结算与奖励分发（胜利条件由 **关卡 / Meta 配置** 决定）             |
| 放弃 / 暂停退出 | 记为未完成；是否给部分元进度由 Meta 规则决定                          |
| 局外资源      | 将本局表现映射为 **金币、经验、解锁进度** 等（**具体公式与章节绑定**见第二部第 22 节） |


**具体时长目标、金币公式、首通规则** 不在第一部展开。

### 3.2 术语


| 术语              | 含义                                                                            |
| --------------- | ----------------------------------------------------------------------------- |
| **一轮遍历**        | 从左到右扫描当前牌序；每步取最长合法组并打出；指针跳过整组                                                 |
| **出牌间隔 / 出牌冷却** | 相邻两次「本步组牌逻辑打出完成」之间的最短等待；受 **玩家基础属性·节律**（第 4.3 节）修饰                            |
| **装配间隔**        | 本轮 **最后一组逻辑打出后** 到 **下一轮从左端扫描** 的等待；受 **玩家基础属性·节律**（第 4.3 节）修饰；**不等待**弹道命中    |
| **单张**          | 见第 5.4 节；本步为单张型时，只触发 **该点数·单张档**（第二部）                                         |
| **牌型**          | 见第 5.4 节；含 **同点数**（对子/三/四）与 **异点数**（顺子/连对/飞机等），分别查第二部 **同点数延伸表** 与 **异点数合成表** |
| **点数效果**        | 每个点数在第二部均有 **单张档**；同点数多张时另有 **延伸档**；**不再使用「关键张」术语**                           |
| **数值乘区回落**      | 某牌型或点数在第二部尚未配置专属表现时，可先经 **基础伤害 + 组牌乘区 + 花色 + 全局档** 等通用链输出（实现可逐步被点数表替换）        |
| **逻辑相 / 表现相**   | 先结算治疗、无敌、资源等**逻辑**，再播放弹道、爆炸、激光、灼地等**表现**                                      |
| **内容 ID**       | 本文敌人、遗物、Boss 使用稳定 ID（如 `E-07`、`R-12`），便于资源与本地化挂载                              |
| **加成标签**        | 某次 **攻击/治疗/护盾** 等技能在结算时 **允许参与乘算** 的玩家属性维度集合；须与第 **4.3.5 节** 标准标签一致，见第二部各技能行  |

## 4. 玩家角色与战斗底台

### 4.1 职责划分


| 功能      | 设计说明                                              |
| ------- | ------------------------------------------------- |
| **移动**  | 八向或虚拟摇杆式连续移动；与障碍、边界碰撞；速度、加速度可经升级修饰                |
| **生命**  | 上限、当前生命；受敌接触或弹道伤害；**治疗与无敌**由牌效管线中的逻辑相提供（具体来源见第二部） |
| **经验**  | 击杀或掉落拾取；达阈值升级触发 **三选一**等入牌决策（具体以进度文档待办为准）         |
| **摄像机** | 跟随玩家；保持战场可读性（不遮挡牌组 UI）                            |


### 4.2 与牌组的关系

**牌组不通过「手动瞄准」直接操作**；玩家通过 **选牌、排序、入牌** 间接控制**出牌顺序与组牌结果**，通过走位控制**生存与拾取**。

### 4.3 玩家基础属性（幸存者向）

本节约定 **单局内** 角色可携带、可被升级/遗物/临时 Buff 修改的 **基础属性维度**（与 **牌面点数效果**、**花色**、**敌人数据** 并列，由管线在结算时合并）。**具体初值、成长曲线与公式系数** 放在 **第二部** 或工程调参表；第一部只保证 **语义一致** 与 **乘区不打架**。

#### 4.3.1 属性分类


| 类别         | 典型属性                                                           | 设计意图                                                 |
| ---------- | -------------------------------------------------------------- | ---------------------------------------------------- |
| **移动与生存**  | 移动速度、**最大生命**、**最大生命加成**、**生命回复**、**护盾**、**防御**、**复活次数**       | 走位、承伤、续航与翻车容错；护盾与生命分流见 **4.3.2**                     |
| **节律与牌组**  | **出牌冷却**（基础出牌间隔）、**装配间隔**（或「装配速度」见下）                           | 控制 DPS 节奏与「空窗期」长度，是扑克驱动幸存者的核心旋钮                      |
| **输出（通用）** | **伤害增加倍率**、**暴击率**、**暴击伤害倍率**、**技能范围加成**、**穿透个数加成**、**弹射次数加成** | 全局修饰牌效产生的攻击面；**哪些技能吃哪几项** 由 **加成标签** 逐条声明（**4.3.5**） |
| **资源与交互**  | **拾取范围**、**等级**、**经验倍率**、**幸运值**、**刷新次数**                      | 升级频率、掉落品质、三选一 **重掷** 与拾取手感；**刷新** 多为 UI/奖励层，一般不进弹道标签 |


**命名对齐**：  

- **出牌冷却** = 两次 **相邻打出组牌**（逻辑步完成）之间的最短等待时间，即第 5 节中的 **基础出牌间隔** 经属性修饰后的结果。  
- **装配速度**：玩家语境中常指「装配阶段多快结束」→ 等价于 **装配间隔缩短**（间隔越小，装配越快）。文档正文优先写 **装配间隔**（秒或逻辑帧），避免「速度越大数字越大」与程序字段符号反着记。

#### 4.3.2 各属性语义（第一部）


| 属性         | 含义                                                             | 修饰来源（举例）                   | 备注                                                                  |
| ---------- | -------------------------------------------------------------- | -------------------------- | ------------------------------------------------------------------- |
| **移动速度**   | 世界单位下单位时间位移量                                                   | 升级、遗物、临时 Buff、花色触发         | 可有上下限，避免不可读或穿模                                                      |
| **出牌冷却**   | 上一组打出完成到 **允许** 打出下一组 的间隔                                      | 全局遗物、局内升级、部分花色/点数（若配置为改节律） | 与 **牌组运行时** 绑定；**不等待** 弹道命中再计下一拍（与第 5 节一致）                          |
| **装配间隔**   | 本轮 **最后一组已打出** 到 **下一轮从左端开始** 的间隔                              | 遗物、升级、环境 debuff            | 「装配加速」= 本值 **降低**                                                   |
| **伤害增加倍率** | 对 **声明了「伤害倍率」加成标签** 的技能，在基底伤害上叠加的全局乘区（或「加法池再转乘」）               | 升级、遗物                      | 与 **暴击** 的先后顺序见 **第 9.5 节**；**未**在标签中声明则 **不参与**                    |
| **技能范围加成** | 对 **爆炸半径、索敌半径、扇角、激光长度** 等「空间尺度」的修饰（按技能标签选择性应用）                 | 同上                         | 非统一「全技能 +X% 半径」：仅 **标签含「范围」** 的技能参算                                 |
| **穿透个数加成** | 投射物 **可额外贯穿** 的目标次数（在基底穿透上 **加法**）                             | 同上                         | 仅对 **弹道类** 且标签含 **「穿透」** 的技能生效                                      |
| **弹射次数加成** | 命中后 **额外弹跳/折射** 次数（在基底弹射上 **加法**）                              | 同上                         | 仅对标签含 **「弹射」** 的技能生效                                                |
| **暴击率**    | 一次 **可暴击的伤害或治疗结算** 触发暴击判定的概率                                   | 遗物、等级、套装类构筑                | 建议硬顶 **100%**；是否对持续伤害、多段跳伤逐段判定由实现约定并在测试文档写清                         |
| **暴击伤害**   | 暴击时相对 **非暴击** 结果的倍率或加算段（如「×1.5」或「+50% 最终」）                     | 同上                         | 与 **点数效果** 的乘区顺序在 **效果管线** 中固定，避免重复乘两次                              |
| **防御**     | 减少受到的最终伤害或提高等效有效生命                                             | 升级、遗物、临时减伤                 | 可用 **护甲公式**（随防御递减收益）或 **百分比减伤+上限**；对 **环境即死**、**真实伤害** 是否生效由第二部标签约定 |
| **生命回复**   | **每秒**（或每逻辑 tick）回复的生命值；与「按最大生命比例的治疗」可叠乘或分流，由技能标签约定            | 升级、遗物                      | **持续回复** 与 **出牌瞬发治疗** 是否同乘区在第二部标清                                   |
| **护盾值**    | **护盾上限** 或 **获得护盾时的基础盾量** 的全局修饰（加法或乘区二选一，全项目统一）                | 升级、遗物、牌效                   | **优先于生命** 扣减；与无敌、霸体的优先级见第二部                                         |
| **最大生命加成** | 在角色 **基底最大生命** 上的 **加法或乘法** 修正（影响 **百分比治疗/伤害（若绑定 MaxHP）** 的基数） | 升级、遗物                      | 与 **当前生命** 上限联动；**复活** 后回满比例另由规则约定                                  |
| **复活次数**   | 单局内 **可触发的额外复活** 剩余次数（死亡时消耗）                                   | 遗物、章节奖励                    | **不属于** 单次攻击/技能的加成标签，由 **规则层** 处理                                   |
| **经验倍率**   | 击杀/拾取经验 **结算乘区**                                               | 遗物、难度修正                    | 一般 **不** 写入弹道技能标签；属 **资源结算层**                                       |
| **幸运值**    | 影响 **掉落品质、权重、暴击球类** 等随机表（具体映射第二部）                              | 遗物、等级                      | 默认 **不** 直接改技能基底伤害；是否影响某技能 **独立** 在内容表声明                            |
| **刷新次数**   | 升级 **三选一 / 四选一** 等界面的 **重掷剩余次数**                               | 遗物、开局 Meta                 | **UI/奖励层**；不进单次出牌伤害管线                                               |
| **拾取范围加成** | 在基底拾取半径上 **加法或乘法** 扩大经验/道具吸附距离                                 | 升级、遗物                      | 与 `CombatTuning` 类基底半径叠乘时约定 **先加后乘** 或 **先乘后加** 之一，全项目唯一            |
| **等级**     | 单局内 **当前角色等级**，驱动升级奖励、部分难度与掉落权重                                | 经验阈值到达时 +1                 | **局外 Meta 等级**（若有）另立字段，勿与局内等级混用                                     |


#### 4.3.3 乘区与平衡原则（不写具体数）

1. **加法池与乘法池分离**：例如移速、拾取半径可设「平坦加成 + 最终乘区」；暴击率多用 **加法叠层再封顶**。
2. **节律类属性慎叠**：出牌冷却与装配间隔若同时被多系统修改，建议 **先加算固定秒数修正，再乘百分比**，或 **按来源优先级** 取主因，避免帧级抖动。
3. **暴击与牌效**：**点数/牌型/花色** 先得出 **基底结算**，再套 **玩家暴击**（或标明某类效果不可暴击）。
4. **防御与无敌**：无敌、护盾、伤害免疫 **优先于** 防御结算；防御 **不改变** 组牌规则与出牌顺序。
5. **伤害倍率与形状**：**伤害增加倍率**、**范围**、**穿透**、**弹射** 仅对 **已声明对应加成标签** 的技能生效，避免「全技能隐性吃满全局」导致平衡失控。
6. **护盾与生命**：护盾扣减先于生命；**生命回复** 与 **百分比治疗** 是否共享同一「治疗加成」标签由第二部按技能行约定。
7. **可读性**：HUD 宜展示 **生命、护盾（若有）、经验/等级、关键节律**；暴击可用飘字或统计面板。

#### 4.3.4 与其它章节的衔接

- **第 5 节**：牌组步进节律的 **基底** 来自工程常量；**玩家属性** 对其做 **玩家向修饰**（冷却、装配）。  
- **第 9 节**：效果管线在 **逻辑相** 合并玩家 **暴击、防御、伤害倍率、范围/穿透/弹射（按标签）** 等与牌面无关或全局的项。  
- **第 11 节**：升级与遗物可发放 **上述任一维度的永久或限时修改**。  
- **第二部**：敌人、关卡、经济表与 **参考玩家 DPS/生存窗口** 对齐时，须声明采用的 **属性基准**（如 1 级白板 vs 满装）；**每条技能须带加成标签**（见 **4.3.5**）。

#### 4.3.5 技能加成标签（攻击 / 技能与玩家属性的绑定）

**目的**：玩家属性池扩大后，**并非** 每一项都作用于每一次结算。须在 **内容设计** 阶段为 **每条攻击或技能效果**（含弹道、爆炸、激光、灼地、治疗包、护盾生成等）**显式声明** 其可吃到的 **玩家属性加成**，并以 **加成标签** 列于第二部表格或技能子表中，供策划、程序、测试 **同一套语言** 对齐。

**标准标签集合**（与 **4.3.2** 可参与「单次技能结算」的维度对齐；命名在工程实现中可映射为枚举，本文不写代码名）：


| 加成标签       | 含义（吃到时）                                              |
| ---------- | ---------------------------------------------------- |
| **伤害倍率**   | 应用玩家的 **伤害增加倍率** 乘区                                  |
| **范围**     | 应用 **技能范围加成**（半径、长宽、扇角等按该技能定义的尺度字段）                  |
| **穿透**     | 应用 **穿透个数加成**（对投射物/贯穿段数类）                            |
| **弹射**     | 应用 **弹射次数加成**（对弹跳/链式类）                               |
| **暴击**     | 参与 **暴击率 / 暴击伤害** 判定（若该技能允许暴击；治疗可单独约定）               |
| **最大生命基数** | **按最大生命比例** 的伤害或治疗，其 **MaxHP** 使用已含 **最大生命加成** 的最终上限 |
| **护盾强度**   | **护盾值** 属性参与本技能产生的护盾量结算                              |
| **生命回复协同** | **生命回复** 属性与本技能的瞬时/持续治疗 **是否叠乘或取高**（由行内一句说明配合）       |


**不写进标签、由系统层单独处理的属性**（一般不出现在弹道行）：**出牌冷却**、**装配间隔**、**移动速度**、**拾取范围**、**经验倍率**、**幸运值**（除非该行明确写「幸运参与权重」）、**刷新次数**、**复活次数**、**防御**（在 **玩家受击** 入口统一结算，而非写在敌受击技能上）。

**填表规则**

1. **每条技能行**（或每个 **命令模板**）须包含一列 **「加成标签」**：列出本技能 **吃到的** 标签，使用上表 **中文名**；可多项，用顿号或分号分隔。
2. **未列出某标签** ⇒ 该技能在对应维度 **不使用** 玩家该属性（按 **0 额外加成** 或 **仅内容基底** 结算）。
3. **迁移期**：先行实装内容若暂未填列，实现侧可 **临时** 按版本说明采用 **默认集合**（例如仅 **伤害倍率 + 暴击**）；**正式母本须补全**，避免长期依赖隐式默认。
4. **与牌效分层一致**：组牌 / 点数 / 花色先给出 **内容基底**，再按标签套 **玩家全局属性**；顺序见 **第 9.5 节**。

#### 4.3.6 `CombatTuning`（难度基底）与角色动态属性（实现侧）


| 层级                                                 | 职责                                                                     | 与玩法的关系                                                                                                       |
| -------------------------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `**CombatTuning` 常量**                              | 关卡与牌面 **难度规划** 用的 **不变** 基底（单发伤害、扇角、地图尺寸、先行点数档等）                       | **不**随拾取改变；工程侧为集中 `const`                                                                                    |
| **玩家动态属性（如 `PlayerCombatStats` → `CombatPlayer`）** | 单局 **实际参战数值**（伤害倍率、技能范围、额外弹道枚数、暴击、承伤、护盾、经验倍率、拾取半径倍率等）                  | **动态**，由拾取/升级/Buff 修改；**开局初值** 由 `CombatTuning` 中 `**PLAYER_DEFAULT_`*** 与场景基底常量统一配置，再写入 `PlayerCombatStats` |
| **效果解析**                                           | 先用 **组牌 + Tuning/内容表** 生成 **命令基底**，再套 **玩家动态属性**，再叠 **花色/遗物/全局强化快照** 等 | 与 **第 9 节** 管线顺序一致                                                                                           |


**结论**：二者 **不冲突**：Tuning 管「纸面难度与内容锚点」，角色属性管「局内实际强弱」。

## 5. 牌组系统：流程、原理与表现

### 5.1 系统职责

牌组系统维护 **当前有序牌列**、**读指针** 与 **时间节律**，在每一逻辑时刻向「组牌检测 → 效果解析」子系统输出 **本步要打出的牌组**；不负责具体伤害数值（由结算与内容表决定）。

### 5.2 运作流程（逻辑顺序）

1. **初始化序列**：开局阶段从牌池生成或玩家选择，得到初始有序列表（长度与规则由 **工程基线 / 第二部关卡规则** 约定）。
2. **步进**：指针从左向右移动；每步根据组牌规则取 **当前最长合法组**。
3. **打出**：将该组交给效果管线；指针跳过整组张数。
4. **节律**：步与步之间受 **基础出牌间隔**（经 **第 4.3 节** 玩家属性修饰后即 **出牌冷却**）约束；当本轮 **最后一组在逻辑上已打出**，开始 **装配间隔**（亦受第 4.3 节修饰），再进入下一轮从左端重新扫描。
5. **局内变更**：升级、奖励等通过 **插入 / 删除 / 重排** 改变序列，从而改变后续所有组牌结果。

**原理**：策略性来自 **序的构造**，而非出牌瞬间的微操；玩家通过选牌、排序、入牌间接控制输出节奏。

### 5.3 表现层（玩家可见）

- **序列可读**：牌带顺序固定展示，即将打出的一组有高亮或聚拢。  
- **时间可读**：出牌与装配有明确动画或音效节拍，**不以弹道命中**作为逻辑等待条件。  
- **状态可读**：装配中可展示「下一轮将从头扫描」等提示，降低学习成本。

**与内容规格的分界**：**同点数延伸**的正式规格（先行已写：**点数 2**）见 **第二部第 17.1 节**；每张牌在 **五十二牌个体** 上的代号、目标基础伤害与风味 **待后续章节补表**。

### 5.4 牌型与单张（概念声明）

本游戏结算只区分两类出牌形态（与「扫描出的 `GroupType`」对应）：


| 概念     | 含义                                                                                 |
| ------ | ---------------------------------------------------------------------------------- |
| **单张** | 本步打出的牌组在组牌检测上为 **单张型**（仅一张，且不与相邻牌构成更大合法牌型）。**直接使用该张牌面点数的「单张档」效果**（第二部 **点数效果规格**）。 |
| **牌型** | 本步打出的牌组为 **两牌及以上**，且满足扑克组牌规则。**牌型** 又分为两类：                                         |


**（1）同点数牌型**：**对子、三条、四条**（整组同一牌面点数）。效果理解为：**该点数在「单张档」基础上的阶跃延伸**——发数、治疗、无敌、激光等随张数升级（第二部以表列出；**不再使用「关键张」一词**，凡点数均可配置单张→对子→三→四的延伸）。

**（2）异点数牌型**：**顺子、连对、飞机** 等跨点数组合。效果理解为：**参与其中的多个点数之单张特质，经牌型规则合成后的强化表现**（第二部 **异点数牌型合成表**）；与「同一点数重复」的延伸不同，侧重 **轨道、段数、范围** 等组合技。

**与旧文档的衔接**：实现中若仍保留「仅部分点数先接入专属弹道」的代码路径，属于 **落地优先级**，不改变上述 **「每点数均可有单张+延伸」** 的产品定义。

## 6. 组牌检测与常规攻击结算系统

### 6.1 组牌检测子系统

- **输入**：有序牌列、当前指针位置。  
- **输出**：本步 **组牌类型**（单张、对子、顺子、连对、飞机等）与 **参与张列表**。  
- **原则**：在合法牌型中取 **张数最长**；并列时按 **牌型优先级**（具体优先级与实现须与本文不变量及组牌检测模块一致）。  
- **不变量**：同一轮遍历内每张牌只属于一个已打出组；顺子长度、A 是否可作 1 等 **以本文与代码注释中的约定为准**（与旧 `RULES_V0_2` 可能相同，但不再单独依赖该文件）。

### 6.2 效果解析主路径（概念）

组牌检测输出 **本步 `GroupType` + 牌列表** 后，解析器按下列 **优先级** 选择效果来源（具体查表均在第二部）：

1. **单张**：`GroupType` 为单张 → **该 rank 的单张档**（点数效果表）。
2. **同点数牌型**：对子 / 三条 / 四条，且整组同 rank → **该 rank 的同点数延伸档**（对子/三/四行），覆盖或叠加于单张语义。
3. **异点数牌型**：顺子 / 连对 / 飞机等 → **牌型合成档**（多 rank 组合规则），不逐张套用单张档。
4. **数值乘区回落**：若某档位在第二部尚未配置专属表现，则回退到 **单卡基础 + 组牌乘区 + 花色 + 全局档 + 遗物**（与下节顺序一致）。

输出统一为 **命令计划**（逻辑相 / 表现相），由战斗执行层消费。

### 6.3 通用叠乘链（回落与补充）

当走 **回落** 或需与专属表现 **叠乘** 时，顺序与第 9 节一致：

1. **单卡基础贡献**（数据层）
2. **组牌类型乘区**（与牌型形状相关）
3. **本次攻击花色层**（第 8 节 + 第二部第 18 节）
4. **同花强化**
5. **全局花色构筑层**
6. **遗物与规则修正**

**与点数专属表现的关系**：专属表现 **可替代** 或 **部分替代** 上述链中的弹道层；治疗、无敌等 **逻辑相** 仍按命令语义执行。**花色对点数专属弹道是否二次修正** 为可选，由版本策略决定。

## 7. 牌型与点数效果（机制层）

### 7.1 设计目标

- **每个牌面点数** 在完全体中均有 **可区分的单张语义**；同点数牌型为该语义的 **强化档位**。  
- **异点数牌型** 为 **多点数语义的合成**，而非「某几张是关键张」的特例。  
- 管线须支持：单张 → 同点数延伸 → 异点数合成 →（可选）通用乘区。

### 7.2 单张路径

检测为 **单张** 时，**只** 读取 **该 rank·单张档**（第二部）；不触发对子/三/四延伸，也不走异点数合成。

### 7.3 同点数牌型路径

检测为 **对子 / 三条 / 四条** 且 **整组同 rank** 时，读取 **该 rank·对应张数的延伸档**（第二部 **§17.x 该点数小节**，先行见 **§17.1 点数 2**）。**延伸** 在语义上必须是 **该点数单张表现的升级**（更多发、更大范围、更高治疗等），具体数值在第二部配置。

### 7.4 异点数牌型路径

检测为 **顺子、连对、飞机** 等时，读取 **牌型合成规格**（第二部 §17.2 异点数牌型合成），按牌型定义综合多个 rank 的贡献（例如按长度、按包含的点数集合加权）。**单张档** 在此路径下 **不单独逐张结算**，除非合成表明确引用（由第二部规定）。

### 7.5 与花色、遗物的衔接

花色本地 / 全局、遗物修正的 **插入顺序** 见第 9 节；对 **点数专属表现** 是否叠乘，在第二部与数值策略中约定（第一部只保留 **可配置槽位**）。

## 8. 花色系统（机制层）

### 8.1 双层计数结构


| 层级       | 作用域     | 系统职责                                          |
| -------- | ------- | --------------------------------------------- |
| **本次攻击** | 当前打出的一组 | 根据组内各花色 **张数分布**，决定是否触发「本地花色效果」及是否触发 **同花强化** |
| **全局构筑** | 当前牌组整体  | 按 **花色独立** 统计牌组内张数，跨多步累计，用于 **分档解锁** 长期加成     |


**原理**：鼓励玩家在 **单步形状**（组内堆叠）与 **整副牌分布**（构筑）之间做权衡。

### 8.2 与结算的衔接

- 本地层与全局层 **均** 参与 **常规攻击链** 的叠乘（顺序见第 9 节）。  
- **阈值档位、倍率、四花色各自偏伤/段数/速度等** — **不在此列举**，见 **第二部第 18 节**。  
- 与 **点数专属表现**（同点数延伸、异点数合成）的叠乘关系：可配置为 **不叠 / 仅叠表现 / 全叠**；由版本与平衡策略决定，第一部只约定 **可插入的槽位**。

## 9. 效果管线与命令模型

### 9.1 分层原则（模块边界）


| 层次        | 职责                             | 禁止                    |
| --------- | ------------------------------ | --------------------- |
| **牌逻辑层**  | 组牌、花色统计、全局强化快照、解析为 **结构化命令计划** | 不直接生成具体节点动画、不写碰撞伤害最终帧 |
| **战斗执行层** | 索敌、生成投射物/激光/爆炸、应用治疗与无敌、飘字      | 不写组牌规则、不判定「这是不是顺子」    |


### 9.2 数据流（概念）

```
一次出牌步：组牌结果 + 牌引用 + 全局花色计数 …
  → 构造「本步上下文」
  → 按牌型/单张选择：单张档 / 同点数延伸 / 异点数合成，或回落到通用叠乘链
  → 产出「命令计划」（含阶段标记）
  → 战斗调度器按阶段执行
```

### 9.3 命令与两相执行

- **逻辑相**：治疗、无敌、资源变更等须**确定性**、与表现无关的结果。  
- **表现相**：弹道、爆炸、激光、灼地等可异步、可叠视觉但**不反向改写**已结算的逻辑结果。

### 9.4 玩家全局属性与管线衔接（概念）

- **节律**（出牌冷却、装配间隔）：由 **牌组运行时** 按玩家当前属性决定「何时允许下一步」；**先于** 本步命令内容，**不改变** 组牌检测结果。  
- **伤害倍率、范围、穿透、弹射**：在 **逻辑相** 对 **本步命令** 结算时，**仅当** 该命令在内容表中声明了对应 **加成标签**（第 **4.3.5** 节）时，才合并玩家侧 **伤害增加倍率**、**技能范围加成**、**穿透个数加成**、**弹射次数加成**。  
- **暴击**：在 **逻辑相** 对「标签含 **暴击** 且允许判定」的结算 **一次** 判定并套用 **暴击伤害**。  
- **治疗 / 护盾**：瞬发治疗、持续回复、护盾生成是否吃 **暴击**、**最大生命基数**、**护盾强度**、**生命回复协同** 等，**以该技能行的加成标签为准**（第 4.3.5 节）。  
- **防御**：在 **玩家受伤害入口** 统一结算；**无敌 / 护盾** 等优先于防御（与第 4.3 节一致）。  
- **拾取范围、经验倍率、幸运、刷新、复活次数**：走 **掉落 / UI / 规则层**，**不** 混入单条弹道命令，除非内容行 **显式** 声明（如幸运参与某随机段数）。  
- 乘区顺序与防重复乘算见 **第 4.3.3 节** 与下文 **9.5** 叠乘链。

### 9.5 与自动攻击的衔接

「牌组步进」与「武器开火」在体验上合一：**由出牌事件驱动**本次攻击效果，而非玩家单独点击攻击键（除非 UI 另有「强制触发」类遗物，属扩展）。

**管线顺序（完全体建议）**：① 单卡基础 → ② 组牌乘区 → ③ 本次攻击花色 → ④ 同花强化 → ⑤ 全局花色档 → ⑥ 遗物与规则修正 → ⑦ **玩家伤害倍率 / 范围 / 穿透 / 弹射**（按本步每条效果的 **加成标签** 选择性应用）→ ⑧ **暴击**（仅标签允许者）→ **敌受击侧** 再算减伤/抗性；**玩家受击** 在独立入口套 **防御 / 护盾**。**百分比生命类** 治疗或伤害若标签含 **最大生命基数**，则 MaxHP 须已含 **最大生命加成**。

## 10. 敌人与战场系统

### 10.1 战场与刷新

- **刷新器** 根据 **关卡段、时间轴、玩家等级** 生成敌人；设 **同屏软上限**，超限优先提高 **精英率** 而非无上限堆怪。  
- **难度曲线**：敌生命 / 伤害随 **时间与章节** 缩放，并与玩家 **构筑强度**（牌组规模、遗物层数）保持张力。  
- **可读性系统**：敌弹与自机弹道 **色相区分**；高威胁攻击配 **预警**（圈、线、吟唱条）。

### 10.2 敌人类型原型（抽象）


| 原型            | 系统职责                     |
| ------------- | ------------------------ |
| **追击 / 近战**   | 持续压缩走位空间                 |
| **远程 / 抛物**   | 强迫离开舒适区，利用掩体与位移          |
| **肉盾**        | 延长接敌时间，检验爆发与段数           |
| **场地 / 控制**   | 改变地面状态或玩家移动，破坏站桩         |

## 18. 花色具体效果、阈值与四花色角色表

### 18.1 阈值与触发（全局 + 本次攻击）

## 22. 关卡结构、难度与局外经济（完全体）

### 22.1 章节结构


| 章   | 主题   | 时长目标      | Boss |
| --- | ---- | --------- | ---- |
| 1   | 废弃赌厅 | 8～10 min  | B-01 |
| 2   | 王牌回廊 | 10～12 min | B-02 |
| 3   | 钟塔核心 | 12～15 min | B-03 |
| 隐藏  | 虚构同花 | 15～20 min | B-04 |


### 22.2 难度阶

- **阶 1～5**：敌人生命 **×(1+0.15×(阶-1))**，伤害 **×(1+0.1×(阶-1))**；高阶增加精英词缀数量。  
- **无尽**：每分钟叠 **+2% 敌生命**，每 5 分钟注入 **新词缀**。

### 22.3 局外经济

- **金币**：`金币 = 基础击杀金 × (1 + 0.001×存活秒) + 组牌奖励`；组牌奖励：`顺子` > `三条` > `对子` > `单张`（具体系数表由数值迭代）。  
- **局外经验**：用于 **解锁角色皮肤、初始遗物池、难度阶**；**不**直接加局内基础攻击（避免膨胀）。  
- **商店（局外）**：用金币购买 **已见遗物** 的「开局携带权」（限 1 件开局）。
